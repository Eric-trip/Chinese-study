# 真题录入规则文档

> 最后更新：2026-06-24
> 维护人：苏木 🌿

---

## 一、整体流程

```
收到原卷 → 识别题型 → 拆分子题 → 打标签 → 对话内复核 → 写入JSON → 推送上线
```

1. **收到原卷**：用户提供 docx/pdf/图片等格式的真实试卷
2. **识别题型**：试卷标注优先 > 题面特征 > 人工判断
3. **拆分子题**：一道大题含多个小题的，拆成独立题目
4. **打标签**：按本文档规则打全标签
5. **对话内复核**：逐题展示给用户，用户确认"全过"或有问题的报编号
6. **写入JSON**：复核通过后写入 `data/exam-questions.json`
7. **推送上线**：commit → push main → 部署 gh-pages

## 二、题目编号规则

**格式：`年份-地区缩写-序号`**

| 组成 | 说明 | 示例 |
|------|------|------|
| 年份 | 4位数字 | 2022 |
| 地区缩写 | 拼音首字母大写 | GX（广西）、BJ（北京）、SH（上海） |
| 序号 | 两位数字，按录入顺序 | 01、02、...17 |

**示例：**
- `2022-GX-BBW-01` → 2022年广西北部湾中考 第1题
- `2024-BJ-03` → 2024年北京中考 第3题
- `2023-SH期末-05` → 2023年上海期末 第5题

**规则：**
- 编号全局唯一，不重复
- 一套试卷一个前缀，序号从 01 递增
- 同一大题的多个小题各自独立编号

## 三、来源标签体系

### 3.1 三档来源（`qsrc` 字段）

| qsrc | 名称 | 定义 | 典型场景 |
|------|------|------|---------|
| `exam` | 真题 | 真实考试场景用过的题 | 中考、期末、月考、期中 |
| `mock` | 模拟题 | 未在真实考试中使用的练习题 | 习题册、教辅、网上题库 |
| `auto` | 自出题 | 代码自动生成的题 | 系统根据数据源实时生成 |

**判断标准：** 题目是否在真实考试场景下使用过。是→exam，否→mock。

### 3.2 来源显示标签（`sourceLabel` 字段）

用于答题界面展示，格式统一：

| 类型 | 格式 | 示例 |
|------|------|------|
| 中考 | `年份·地区中考` | `2022·广西北部湾中考` |
| 期末 | `年份季·年级期末` | `2023秋·七年级期末` |
| 月考 | `年份月·月考` | `2024·10月月考` |
| 习题册 | `书名` | `XX习题册` |
| 教辅 | `书名` | `初中语文必刷题` |

### 3.3 字段命名注意事项

- **题目来源字段用 `qsrc`，不用 `source`**
- 原因：古诗文数据中已有 `source` 字段表示诗句出处（如 `《岳阳楼记》`），用 `qsrc` 避免撞车
- `sourceLabel` 是显示用的人类可读标签
- `auto` 来源的题目由生成器代码自动打 `qsrc: 'auto'`，不需要 `sourceLabel`

## 四、题目数据结构

每道题的 JSON 结构：

```json
{
  "id": "2022-GX-BBW-01",
  "paper_id": "2022-GX-BBW",
  "original_no": 1,
  "type": "pinyin_char",
  "typeName": "字音字形",
  "format": "fill",
  "difficulty": "easy",
  "qsrc": "exam",
  "sourceLabel": "2022·广西北部湾中考",
  "status": "approved",
  "question": "题干文本",
  "options": [],
  "answer": "答案",
  "explanation": "解析",
  "knowledgePath": "基础知识 > 语音"
}
```

### 4.1 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | ✅ | 全局唯一编号 |
| `paper_id` | ✅ | 所属试卷ID |
| `original_no` | ✅ | 原卷中的题号 |
| `type` | ✅ | 题型ID（见第五节） |
| `typeName` | ✅ | 题型显示名 |
| `format` | ✅ | 题目形式：`choice`/`fill`/`truefalse` |
| `difficulty` | ✅ | 难度：`easy`/`medium`/`hard` |
| `qsrc` | ✅ | 来源：`exam`/`mock`/`auto` |
| `sourceLabel` | exam/mock 必填 | 来源显示标签 |
| `status` | ✅ | 状态：`pending`/`approved` |
| `question` | ✅ | 题干文本（`\n` 换行） |
| `options` | choice 必填 | 选项数组，每项 `{text, correct}` |
| `answer` | ✅ | 正确答案 |
| `explanation` | ✅ | 解析说明 |
| `knowledgePath` | 可选 | 关联知识点路径 |

### 4.2 status 状态门

| 状态 | 含义 | 练习能否抽到 |
|------|------|-------------|
| `pending` | 待复核 | ❌ 不会出现在练习中 |
| `approved` | 已通过复核 | ✅ 正式入库，可被练习抽到 |

- 新录入的题默认 `pending`，对话内复核通过后改为 `approved`
- 练习系统只抽 `approved` 的题

## 五、题型对照表

### 5.1 已注册题型（`data/question-types.json`）

| type ID | typeName | category | auto_generate | 说明 |
|---------|----------|----------|---------------|------|
| `pinyin` | 字音 | basic | ✅ | 判断题为主 |
| `char` | 字形 | basic | ✅ | 选择题 |
| `pinyin_char` | 字音字形综合 | basic | ❌ | 填空题，真题录入 |
| `word_usage` | 词语运用 | basic | ✅ | 含旧ID `idiom` |
| `rhetoric` | 修辞手法 | basic | ✅ | 判断题+选择题 |
| `literature` | 文学常识 | basic | ✅ | 选择题 |
| `recitation` | 古诗文默写 | basic | ✅ | 填空题，含旧ID `quote` |
| `sentence` | 病句辨析 | basic | ✅ | 选择题 |
| `punctuation` | 标点符号 | basic | ❌ | 仅真题 |
| `culture` | 文化常识 | basic | ❌ | 仅真题 |
| `classics_reading` | 名著/文言文阅读 | classics | ❌ | 仅真题 |
| `language_practice` | 语言运用 | integrated | ❌ | 仅真题 |

### 5.2 旧题型ID兼容映射

| 旧 ID | 新 ID | 说明 |
|-------|-------|------|
| `idiom` | `word_usage` | 成语 → 词语运用 |
| `quote` | `recitation` | 名句默写 → 古诗文默写 |

错题本中旧ID记录不会丢失，加载时自动映射。

### 5.3 题型识别规则

录入真题时，按以下规则识别题型：

1. **试卷标注优先**：原卷标题写了"字音""字形""病句"等，直接用
2. **题面特征判断**：
   - "给加点字注音"/"根据拼音写汉字" → `pinyin_char`
   - "下列词语中没有错别字"/"找出错别字" → `char`
   - "病句"/"没有语病" → `sentence`
   - "补写诗句"/"默写" → `recitation`
   - "赏析有误"/"理解正确" → `classics_reading`
   - "翻译句子" → `classics_reading`
   - "断句" → `classics_reading`
   - "古义/今义" → `classics_reading`
3. **人工判断**：以上都无法确定时，根据题目内容人工归类

## 六、各题型录入规范

### 6.1 字音字形（pinyin_char）

**题干格式：**
- 注音题：用【】标记要注音的字，如 `【沧】海桑田`
- 写汉字题：保留拼音，如 `春寒liào峭`
- 末尾加示例：`（示例：zhong1 一声，数字表示声调）`

**答案格式：**
- 注音题：拼音+声调数字，如 `cang1`、`liao4`
- 写汉字题：直接写汉字，如 `料`

**解析格式：**
- 先说本题答案，再补充成语/词语全文
- 示例：`"沧"读作 cāng（一声）。沧海桑田（cāng hǎi sāng tián），比喻世事变化很大。`

### 6.2 古诗文默写（recitation）

**题干格式：**
- 给出提示语（修辞手法、内容描写、情感等）
- 用全角空格 `　　` 表示填空
- 如需提示出处，在括号中注明

**示例：**
```
根据提示填写诗文（毛泽东《沁园春·雪》）：

互文，写寒冬雪景，气势磅礴，境界阔大。
　　，　　。
```

**答案格式：**
- 完整诗句，用逗号分隔
- 如：`千里冰封，万里雪飘`

### 6.3 病句辨析（sentence）

**题干格式：**
- 选择题：`下面句子中，属于病句的一项是（　）`
- 选项用 `A.` `B.` `C.` `D.` 开头

**答案：** 选项字母，如 `A`

**解析：** 说明病句类型和修改方法，如 `A项成分残缺，缺主语，可在"往往"前加上"人们"。`

### 6.4 文言文阅读（classics_reading）

涵盖：文言文选择、翻译、断句、古今异义等。

**翻译题：**
- 题干：`将文中画横线的句子翻译成现代汉语：\n\n原句`
- 答案：现代汉语翻译
- 解析：逐字解释关键字

**断句题：**
- 题干：`用"/"给下面的句子断句（限断两处）：\n\n原句`
- 答案：`义熙二年/解印去县/乃赋《归去来》`
- 解析：说明断句依据

**古今异义题：**
- 题干：列出词语，要求填写古义和今义
- 答案：逐词列出，用分号分隔

### 6.5 古诗词鉴赏（classics_reading）

**题干：**
- 包含完整诗词原文
- 问题：`下面对该诗赏析有误的一项是（　）`

**选项：** 4个选项，其中1个有误

**答案：** 选项字母

**解析：** 说明该选项错在哪里，正确应该是什么

### 6.6 选择题通用规范

```json
"options": [
  {"text": "A. 选项内容", "correct": true},
  {"text": "B. 选项内容", "correct": false},
  {"text": "C. 选项内容", "correct": false},
  {"text": "D. 选项内容", "correct": false}
]
```

- 选项前缀统一用 `A.` `B.` `C.` `D.`
- `correct` 字段标记正确选项
- 选项顺序按原卷顺序

### 6.7 填空题通用规范

- 题干中用 `　　` 或 `______` 表示空位
- 答案直接填写正确内容
- 多个空的答案用换行或分号分隔
- 填空题不需要 `options` 字段

## 七、不予录入的题目类型

以下题目不录入题库：

| 类型 | 原因 |
|------|------|
| 开放性题（谈感悟、提建议等） | 无标准答案 |
| 对联配对/排序 | 格式过于复杂 |
|作文 | 无法自动判分 |
| 需要长原文的阅读理解 | 题干过长，暂不支持 |

**说明：** 阅读理解题如果原文不长（如古诗鉴赏、文言文节选），可以连同原文一起录入题干。如果原文超过500字，暂不录入。

## 八、判分规则

### 8.1 填空题判分（submitFill）

**归一化处理：**
1. 去除首尾空格
2. 去除标点符号（中英文逗号、句号、感叹号、问号）
3. 声调符号 → 声调数字（`ā`→`a1`、`á`→`a2`、`ǎ`→`a3`、`à`→`a4`）
4. 转小写

**规则：** 用户输入归一化后 === 标准答案归一化后 → 正确

**拼音题特殊规则：**
- `cang1`（数字声调）✅ 可匹配
- `cāng`（符号声调）✅ 可匹配（归一化为 cang1）
- `cang`（无声调）❌ 不匹配（无声调=轻声，不是一声）
- 声调符号对照表：

| 声调 | 符号 | 数字 |
|------|------|------|
| 一声 | ā ē ī ō ū ǖ | 1 |
| 二声 | á é í ó ú ǘ | 2 |
| 三声 | ǎ ě ǐ ǒ ǔ ǚ | 3 |
| 四声 | à è ì ò ù ǜ | 4 |

### 8.2 选择题判分（submitOption）

- 用户选中的选项 `correct === true` → 正确
- 无模糊匹配，精确判断

## 九、对话内复核流程

### 9.1 录入后展示格式

每道题按以下格式展示给用户：

```
编号｜题型｜形式｜难度
题干内容
答案
解析
来源标签
```

### 9.2 用户操作

- **全部通过**：用户说"全过"→ 所有题写入 JSON，status 设为 `approved`
- **有问题**：用户报编号 + 问题描述 → 修改后重新展示 → 确认后写入

### 9.3 已录入题目的修改

如需修改已入库的题目：
1. 用户报编号（如 `2022-GX-BBW-03`）
2. 在 `data/exam-questions.json` 中找到对应 `id` 的题目
3. 修改对应字段
4. 推送更新

## 十、试卷数据结构

每套试卷在 `papers` 数组中：

```json
{
  "id": "2022-GX-BBW",
  "year": 2022,
  "region": "广西北部湾",
  "examType": "中考",
  "title": "2022年广西北部湾经济区初中学业水平考试",
  "fullScore": 120,
  "duration": 150,
  "qsrc": "exam",
  "sourceLabel": "2022·广西北部湾中考"
}
```

- `id`：试卷唯一标识，与题目编号前缀一致
- `examType`：`中考`/`期末`/`月考`/`期中`
- 题目通过 `paper_id` 关联到试卷

## 十一、缓存版本号规则

每次修改 JS/CSS 文件后，更新 `practice.html` 中的缓存版本号：

- 格式：`v=YYYYMMDDx`（日期+字母序号）
- 示例：`v=20260623a` → `v=20260623b` → `v=20260623c`
- 同一天多次修改递增字母
- 跨日期重置字母为 `a`

**需更新的文件引用：**
- `css/style.css?v=...`
- `js/data.js?v=...`
- `js/question-bank.js?v=...`
- `js/question-registry.js?v=...`
- `js/practice.js?v=...`

## 十二、部署流程

```bash
# 1. 推送 main 分支
git add -A
git commit -m "feat: 描述"
git push origin main

# 2. 部署到 gh-pages
git checkout gh-pages
git pull --rebase origin gh-pages
git checkout main -- .
git add -A
git commit -m "deploy: 描述"
git push origin gh-pages

# 3. 切回 main
git checkout main
```

**注意事项：**
- 推送前先 `git pull --rebase`，避免冲突
- 用户可能自行推送到远程，动手前必须先拉取最新
- gh-pages 远程有新提交时：`git reset --hard origin/gh-pages` 后重新 `checkout main -- .`
