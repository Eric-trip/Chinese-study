## 五、关键设计决策

1. **纯静态网站**：无后端依赖，GitHub Pages 免费托管，同学不需要服务器
2. **题型注册制**：加题型只改 JSON 不改代码，方便扩展
3. **扁平数据结构**：直接用原书 JSON 不转换，避免数据丢失
4. **localStorage 存储**：错题/收藏/笔记全在本地，无需账号
5. **gzip 压缩传输**：130 万字符的 JSON 通过 gzip 压缩后传输，不影响加载速度
6. **图片暂不渲染**：265 张图片文件不在手，不显示占位符
7. **qsrc 字段命名**：避免与古诗文 source 撞车
8. **旧题型 ID 兼容映射**：保证已存储的错题记录不丢失

## 八、架构变更记录

### Day 6 — 2026-06-24（出题策略变更）

**核心变更：彻底放弃实时生成，只使用预制题和真题**

⚠️ **本日经历了两次策略迭代：**
1. 首次（Day 6 上午）：从"纯实时生成"改为"预制题优先，实时生成为辅"
2. **最终（本日确认）：彻底放弃实时生成，只使用预制题和真题**

#### 背景
- 之前的出题引擎（question-bank.js GENERATORS）在用户点击"开始练习"时实时随机生成题目
- 实时生成的题目质量不稳定（模板化解析、干扰项不合理、好题无法沉淀）
- 标点符号、文化常识、名著阅读等题型因无生成器而一直空着，转向预制题后已全部覆盖
- 经过评估，认为运行时实时生成的性价比不高，决定彻底转向预制题 + 真题

#### 新架构

```
               ┌──────────────────────┐
               │    scripts/           │
               │  generate-auto-       │

                          ↓

               │ .json                 │

                          ↓

               │ .json                 │
```

   - `_sampleFromExam()` 只取 `qsrc === 'exam'` 的题

#### 质量保障体系

1. **源数据锚定**：每道题标注 `handbookSource`，可追溯到 handbook 原文
2. **自动校验**：出题后跑 `scripts/validate-auto-questions.js` 检查格式/选项/答案/文字
3. **AI 对抗性复核**：入库后逐题复核（不抽样），四维对抗检查（答案/干扰项/解析/数据源），每道题单独判定 ✅/⚠️/❌，发现问题即修正。详见 `docs/出题规范文档.md` 第六章
4. **用户反馈**：答题界面可标记"好题/有问题"
5. **持续扩充**：题量不足时通过 `scripts/generate-auto-questions.js` 扩充预制题，而非降级到实时生成

## 九、用户偏好备忘

- 偏好一次性全面修复，而非头痛医头
- 会自行改代码（紧凑网格、缓存版本号、紧凑标签流式布局），动手前必须先 pull
- 不喜欢跳来跳去的动画，偏好安静的淡入
- 对出题质量要求高，会逐题审查内容正确性
- 不喜欢长篇讨论，倾向"你觉得怎么做就怎么做"的信任模式
- 对"积分消耗"敏感，与其反复打补丁不如重写
- 会自行分析数据格式问题，提出明确修改方向
- 繁体字敏感，发现即要求修正
