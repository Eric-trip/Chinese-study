### 数据流

```
handbook.json (130万字)
    ↓ fetch + gzip
data.js buildIndex()
    ↓ 构建三级索引（编→部→节）
knowledge.js renderContent()
    ↓ 按节点 type 分发渲染
    ├── heading → h3/h4/h5
    ├── paragraph → content-text
    ├── table → compact-grid / data-table
    └── image → 跳过（265张暂不渲染）

question-bank.js GENERATORS
    ↓ 从 handbook 数据中提取题目素材
    ↓ generateQuestions() 生成 + 质检 + 去重
question-registry.js generateUnifiedQuestions()
    ↓ 合并真题库 + 自主题
practice.js → 答题界面
```

- 病句库 12 类 25 条逐条人工审核

## 五、关键设计决策

1. **纯静态网站**：无后端依赖，GitHub Pages 免费托管，同学不需要服务器
2. **题型注册制**：加题型只改 JSON 不改代码，方便扩展
3. **扁平数据结构**：直接用原书 JSON 不转换，避免数据丢失
4. **localStorage 存储**：错题/收藏/笔记全在本地，无需账号
5. **gzip 压缩传输**：130 万字符的 JSON 通过 gzip 压缩后传输，不影响加载速度
6. **图片暂不渲染**：265 张图片文件不在手，不显示占位符
7. **qsrc 字段命名**：避免与古诗文 source 撞车
8. **旧题型 ID 兼容映射**：保证已存储的错题记录不丢失

## 六、部署流程

```bash
# 1. 推送 main 分支
git push origin main

# 2. 切到 gh-pages，覆盖内容
git checkout gh-pages
git checkout main -- .
git add -A
git commit -m "deploy: 更新说明"
git push origin gh-pages

# 3. 切回 main
git checkout main
```

**注意事项**：
- 推送前先 `git pull --rebase`，避免冲突
- 同学也会自行推送到远程，动手前必须先拉取最新
- gh-pages 远程有新提交时：`git reset --hard origin/gh-pages` 后重新 `checkout main -- .` 再 force push
- GitHub PAT 无 workflow scope，不能推送 .github/workflows/
