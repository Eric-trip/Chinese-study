# 题库框架重设计方案

> 目标：重新设计题型划分，使其同时满足两个维度的对齐——
> ① 与网站知识点（基础知识 12 个 part）一一对应；
> ② 与中考语文"积累与运用"板块的题型自然衔接。
> 同时支持「中考真题」和「自主出题」两种题目来源。

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **知识点对齐** | 每种题型对应手册中一个明确的 knowledge part，不遗漏、不重叠 |
| **中考衔接** | 题型命名、出题形式、考查角度参照中考真题，做到"练什么考什么" |
| **双来源支持** | 真题（人工录入）和自主题（代码生成）共存于同一框架，用户可选来源练习 |
| **向下兼容** | 现有 7 种题型数据可平滑迁移到新框架，不丢失已生成的错题记录 |
| **可扩展** | 框架预留阅读理解、写作等后续板块的扩展空间 |

### 2.2 各题型详细设计

#### 1. 字音
- **中考考法**：给 4 组词语，选出读音全部正确/有误的一项
- **自主出题**：
  - 选择题：4 个词语，3 个读音正确 1 个错误（或反过来），选出有误/正确的一项
  - 判断题：给词语标注拼音，判断是否正确
- **数据源**：handbook「语音 > 必考知识梳理」易错词语表 + 易读错成语表
- **真题入库**：直接录入中考真题选择题

#### 2. 字形
- **中考考法**：给 4 组词语，选出书写全部正确/有错的一项
- **自主出题**：
  - 选择题：4 个词语，含/不含错别字
  - 判断题：判断词语书写是否正确
- **数据源**：handbook「汉字 > 必考知识梳理」易错字表 + 成语中的易错字
- **真题入库**：直接录入中考真题

#### 3. 字音字形综合（新增）
- **中考考法**：给 4 个选项，每个选项含"字音+字形"两项，选出全对的一项
- **自主出题**：从字音库和字形库各取数据，组合成 A/B/C/D 四组（每组含一个词语+拼音标注），3 组正确 1 组有误
- **数据源**：复用字音 + 字形数据

#### 4. 词语运用（改造）
- **中考考法**：
  - ① 给一段话，选最恰当的成语/词语填空
  - ② 给一个含成语的句子，判断成语使用是否正确
  - ③ 选出成语使用正确/错误的一项
- **自主出题**：
  - 选择题：给成语造句，判断使用是否正确（正确句 vs 错误句）
  - 判断题：判断含成语的句子使用是否得当
- **数据源**：handbook「词语 > 必考知识梳理」成语释义
- **说明**：原"成语"题型只考"释义匹配"，改为"运用判断"，更贴合中考

#### 5. 病句辨析
- **中考考法**：选出没有/有语病的一项（4 选 1）
- **自主出题**：
  - 选择题：4 个句子，选没有/有语病的
  - 判断题：判断句子是否有语病
- **数据源**：handbook「句子」病句分类库 + 正确句子库
- **真题入库**：录入中考真题病句题

#### 6. 标点符号（新增）
- **中考考法**：选出标点使用正确/错误的一项
- **自主出题**：
  - 选择题：4 个句子，判断标点用法
  - 判断题：判断句子标点是否正确
- **数据源**：需在 handbook「标点符号」中补充典型例句库（正确句 + 错误句）
- **初期方案**：先录入真题，自主出题待数据补充后启用

#### 7. 修辞手法
- **中考考法**：判断修辞手法 / 分析修辞效果
- **自主出题**：
  - 选择题：给例句选修辞手法
  - 判断题：判断修辞标注是否正确
- **数据源**：handbook「修辞」修辞类型表 + 例句库

#### 8. 文学常识
- **中考考法**：作家作品配对、朝代判断、文学体裁判断
- **自主出题**：
  - 选择题：作家配朝代 / 作品配作者
  - 判断题：判断文学常识表述是否正确
- **数据源**：handbook「文学常识」作家表 + "第一"作品表

#### 9. 文化常识（新增）
- **中考考法**：判断传统文化常识表述是否正确
- **自主出题**：
  - 选择题：给表述选对错 / 给概念选解释
  - 判断题：判断文化常识表述
- **数据源**：handbook「文化常识」附录
- **初期方案**：先录入真题，自主出题待数据补充后启用

#### 10. 古诗文默写
- **中考考法**：
  - ① 上下句默写（给上句填下句 / 给下句填上句）
  - ② 理解性默写（给提示语，写出符合条件的名句）
- **自主出题**：
  - 填空题（上下句默写）— 已有
  - 选择题（理解性默写）— **新增**：给提示语，选符合的名句
- **数据源**：handbook「名句名段」+ 内置名句库

#### 11. 名著阅读（新增）
- **中考考法**：判断人物/情节是否匹配、选择名著相关表述
- **自主出题**：
  - 选择题：给情节判断出自哪部名著 / 给人物选作品
  - 判断题：判断名著情节表述是否正确
- **数据源**：handbook「名著导读」必考知识梳理
- **初期方案**：先录入真题，自主出题待数据补充后启用

#### 12. 语言运用（新增）
- **中考考法**：对联匹配、仿写句子、信息提取与概括、情境转述
- **自主出题**：
  - 选择题：对联上下联匹配、仿写选择
  - 填空题：补写对联/仿写
- **数据源**：handbook「口语交际」+「综合性学习」
- **初期方案**：先录入真题，自主出题待数据补充后启用

## 三、题目形式维度

统一为 4 种形式，兼容现有代码：

| 形式 | 标识 | 说明 | 现有支持 |
|------|------|------|---------|
| 选择题 | `choice` | 4 选 1，含题干+4 个选项 | ✅ 已有 |
| 判断题 | `truefalse` | 2 选 1（正确/错误） | ✅ 已有 |
| 填空题 | `fill` | 文本输入，精确匹配判分 | ✅ 已有 |
| 多选题 | `multi` | 多个选项可选（预留，暂不启用） | ❌ 新增 |

---

## 四、题目来源维度

| 来源 | 标识 | 说明 | 数据量预期 |
|------|------|------|-----------|
| 自主生成 | `auto` | 代码从 handbook 数据自动生成 | 理论上无限 |
| 中考真题 | `exam` | 人工录入的历年真题 | 按年份逐年补充 |
| 模拟题 | `mock` | 人工录入的模拟题/改编题 | 按需补充 |

用户在练习时可选来源：
- **全部**（默认）：真题 + 自主题混合
- **仅真题**：只做中考真题
- **仅自主题**：只做自动生成的题

## 五、数据结构设计

### 5.1 统一题目数据结构

所有题目（无论来源）统一为以下结构：

```javascript
{
  id: "exam_2024_bj_01",        // 唯一ID，真题用 exam_年份_地区_序号，自主题用 auto_题型_序号
  type: "pinyin",                // 题型标识（见 2.1 编号对应 key）
  typeName: "字音",              // 题型显示名
  format: "choice",             // 题目形式：choice | truefalse | fill | multi
  difficulty: "medium",         // 难度：easy | medium | hard
  source: "exam",               // 来源：auto | exam | mock
  sourceLabel: "2024·北京中考", // 来源显示标签（真题显示年份+地区）
  question: "下列词语中读音全部正确的一项是：",
  options: [                    // 选择题/判断题的选项
    { text: "...", correct: false },
    { text: "...", correct: true },
    ...
  ],
  answer: "...",                // 正确答案（填空题为文本，选择题/判断题为选项文本）
  explanation: "...",           // 解析
  isFill: false,                // 是否为填空题（兼容现有代码）
  knowledgePath: "基础知识 > 语音",  // 对应手册路径（可选，用于关联知识点）
  tags: ["多音字", "易混读音"]   // 知识点标签（可选，用于精准练习）
}
```

### 5.2 真题库数据结构

真题以 JSON 文件存储，按年份/地区组织：

```
data/exam-questions/
├── 2024/
│   ├── beijing.json      // 2024年北京中考
│   ├── shanghai.json     // 2024年上海中考
│   └── ...
├── 2023/
│   ├── beijing.json
│   └── ...
└── index.json            // 索引文件，记录所有真题文件
```

单个真题文件格式：
```json
{
  "year": 2024,
  "region": "北京",
  "examType": "中考",
  "questions": [
    {
      "id": "exam_2024_bj_01",
      "type": "pinyin",
      "format": "choice",
      "difficulty": "medium",
      "source": "exam",
      "sourceLabel": "2024·北京中考",
      "question": "下列词语中读音全部正确的一项是（  ）",
      "options": [
        { "text": "A. 憎恶(zèng) 酝酿(niàng)", "correct": false },
        { "text": "B. 憎恶(zēng) 酝酿(niàng)", "correct": true },
        { "text": "C. 憎恶(zèng) 酝酿(liàng)", "correct": false },
        { "text": "D. 憎恶(zēng) 酝酿(liàng)", "correct": false }
      ],
      "answer": "B. 憎恶(zēng) 酝酿(niàng)",
      "explanation": "憎恶读 zēng wù，酝酿读 yùn niàng。",
      "knowledgePath": "基础知识 > 语音"
    }
  ]
}
```

## 六、向下兼容方案

### 6.1 题型映射

| 旧题型 key | 新题型 key | 变化 |
|-----------|-----------|------|
| pinyin | pinyin | 不变 |
| char | char | 不变 |
| idiom | wordUsage | **改名**，考查角度从"释义"改为"运用" |
| rhetoric | rhetoric | 不变 |
| literature | literature | 不变 |
| quote | quote | 不变（增加理解性默写选择题型） |
| sentence | sentence | 不变 |

### 6.2 错题本兼容

现有错题本中存储的 `type` 字段值不变（pinyin/char/idiom/...），加载时做映射：
- 旧错题 `type: 'idiom'` → 显示为新题型名"词语运用"，但内部 key 保持 'idiom' 兼容
- 或在加载时统一映射 `idiom → wordUsage`

### 6.3 分阶段实施

| 阶段 | 内容 | 工作量 |
|------|------|--------|
| **Phase 1** | 真题库基础设施：数据结构、加载逻辑、真题录入工具 | 中 |
| **Phase 2** | 新增题型：标点符号、文化常识、名著阅读、语言运用（先只支持真题） | 中 |
| **Phase 3** | 改造成语题型为"词语运用"（考查角度变化） | 小 |
| **Phase 4** | 新增"字音字形综合"题型 | 小 |
| **Phase 5** | 古诗文默写增加"理解性默写"选择题 | 小 |
| **Phase 6** | 新增题型支持自主出题（待数据补充后逐步启用） | 大 |

## 七、练习界面变化

### 7.1 配置面板新增"来源"选择

```
题型：[全部] [字音] [字形] [词语运用] [病句辨析] ...
难度：[混合] [简单] [中等] [困难]
来源：[全部] [仅真题] [仅自主题]    ← 新增
题量：[10题] [20题] [30题]
```

### 7.2 真题来源标签显示

答题界面中，题目卡片上显示来源标签：
- 真题：显示 `📝 2024·北京中考` 标签
- 自主题：显示 `🤖 自动生成` 标签或不显示

### 7.3 新增"真题套卷"模式

在现有 4 种练习模式（按知识点/随机/模拟/错题）基础上，新增第 5 种：
- **真题套卷**：选择年份+地区，按真题原卷顺序做题（还原真实中考体验）

```
模式选择：
  📌 按知识点练习
  🎲 随机抽题
  📝 模拟测试
  ❌ 错题重练
  📋 真题套卷    ← 新增
```
